# 01 — Legal & Licensing

## IP strategy — zero legal exposure
- **Zero** use of Blizzard/any studio assets, code, item/spell tables, world geometry, names, or lore. Full original IP.
- **Mechanics/systems are not copyrightable** (only specific expression is) — WoW-like systems (talent trees, raids, factions) are legally safe to emulate. We only need original *expression*: art, text, code, data.
- All maps/models/textures/audio: **AI-generated from scratch**, never extracted/ripped from any game client.
- Original naming for everything: factions, races, classes, zones, items, spells, lore.

## License — MIT
- **MIT** for engine + client. Full permissive fork rights, commercial use allowed, no forced contribution.
- No paid tier, no dual-licensing. Pure open source.
- **Governance = goodwill, not copyleft.** We ask operators to upstream meaningful fixes/changes — not enforced via AGPL. Bet on being the best place to build games, not legal mechanisms.
- **Contributor process:** standard PR-based, no CLA. Keep the barrier to contribution low, consistent with MIT.

## Trademark
- Register project **name + logo** as a trademark even though the code is MIT — prevents brand confusion / hostile forks using our name, while keeping the code fully open.

## Why this matters for "best open core"
Every prior WoW-like open core (AzerothCore/TrinityCore) carries latent IP risk because it emulates Blizzard data. An **original-IP, MIT** core is the first one an operator can build a real commercial product on without legal fear. That legal cleanliness *is* part of the moat.
